**Animation contract** (GSAP + ScrollTrigger in a per-page `<script>`):
- Register the plugin; run an **idempotent** `init()` guarded by the standard `data-gsapInit` flag (`if (!root || root.dataset.gsapInit) return;`, then set `root.dataset.gsapInit = '1'` — reuse this exact flag, don't invent a per-page variant); wrap motion in `gsap.matchMedia('(prefers-reduced-motion: no-preference)')`. **Reduced-motion returns early → the page is fully visible and static.**
- Set initial hidden states in **JS, never CSS**, so a no-JS / pre-hydration page still shows all content. Standard hooks: `data-reveal` (fade-up on scroll), `data-stagger` (stagger a group's children), `data-hero-*` (hero entrance timeline).
- **If a CSS rule must own an element's opacity afterwards (e.g. a dim/spotlight state), don't tween `autoAlpha`/`opacity` on it** — a `gsap.from(..., {autoAlpha: 0})` leaves inline `opacity: 1` behind that silently defeats the CSS forever (2026-07-05 review lesson; the puppy page's class-night steps use an x-only entrance for this reason — the day/regime-step dim rules on Day School / Intensive / B&T still carry the no-op, decide at polish).
- ClientRouter lifecycle: re-run `init()` on `astro:page-load`, and **kill all ScrollTriggers on `astro:before-swap`** or they leak across navigations.
- **ScrollTrigger `pin` (only `WholeDogFunnel` uses it):** pin `start` must clear the fixed 76px header (`'top <headerH>px'`, stage sized `calc(100svh - var(--fnl-top))`), the pin must be gated on width **and** height in the CSS media query *and* `gsap.matchMedia` *and* a runtime content-fits check, the matchMedia complement must be `not all and (…)` (a hand-written `max-*` pair leaves fractional viewports matching neither), and **never set `invalidateOnRefresh`** on a timeline whose tweens drive plain proxy objects — it re-records their start values from the current on-screen state.
- **⚠️ You cannot screenshot a scrubbed animation from a backgrounded automation tab:** a hidden tab gets **zero** rAF frames, so GSAP's ticker is frozen and ScrollTrigger never updates — `window.scrollTo` proves nothing. Expose the timeline behind `import.meta.env.DEV`, run `npx astro dev`, and drive `progress()` synchronously. For a real mobile viewport when `resize_window` no-ops, mount the page in a 390px-wide `<iframe>` (media queries follow the iframe's viewport).
- Hero video: `preload="none"`, lazy `.load()/.play()` after first paint, **skipped for reduced-motion or `navigator.connection.saveData`**; the `poster` is the fallback frame.
